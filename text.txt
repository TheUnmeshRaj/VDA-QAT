1. Which metric are we choosing the best model on?
--------------------------------------------------
We are choosing the best model based on ABS_REL (Absolute Relative Error).

In monocular depth estimation, ABS_REL is the master metric because it measures 
the percentage of depth error at every single pixel. By optimizing for ABS_REL, 
we ensure the model gets the geometry of the entire scene—both near and far—
physically correct and scale-aligned.

  these parameters in real-time. Epoch 13 starts with the EXACT, highly-optimized 
  weights that Epoch 12 finished with. No file reloading is needed between epochs!
  
* Individual Epoch Backups: We STILL save a full backup of the weights on disk 
  for every single epoch! That is why you saw the log print:
  "[Ckpt] -> /media/rvcse22/CSERV/vdaproj/checkpoints/vda_qat_epoch012.pt"
  Your Epoch 12 model was fully saved to disk.
  
* The "Best" File is Just a Copy: The "vda_qat_best.pt" file is purely a 
  static, read-only copy of the absolute best checkpoint, kept safe for your 
  final production evaluation. The active, running training loop NEVER reads 
  from "vda_qat_best.pt".

So, the next epoch doesn't need to load a file to know to change something—it is 
already holding the updated weights in the GPU memory and continuing the gradient 
descent smoothly!
